# 班级电子宠物平台 - 需求说明书 (PRD)

## 1. 项目概述
“班级电子宠物平台”是一款结合了游戏化元素的班级管理工具。通过为每位学生分配一个虚拟“电子宠物”，将现实中的班级表现（如纪律、学习、劳动等）转化为宠物的经验值（经验条）。宠物随着积分累计能够升级并“开盲盒”式地获得特殊外观特效，从而激发学生的积极性。同时，平台提供互动大屏和数据面板，方便教师管理和班级文化展示。

## 2. 核心用户角色
- **教师 (Admin)**：管理学生名单，依据学生表现进行加扣分，管理后台数据。
- **学生 (User/Viewer)**：查看自己的宠物状态，点击大屏与宠物互动、聊天。

## 3. 核心页面规划

### 3.0 【产品改进】统一入口门户页 (Portal)
- **定位**：解决多页面导航混乱问题。提供一个清晰的首屏，提供“进入大屏模式”、“教师登录”、“学生查询入口”三个大按钮，明确分流用户。

### 3.1 班级大屏展示页 (大屏页面)
- **定位**：运行在班级多媒体大屏或投影仪上，作为日常展示。
- **UI布局**：
  - 以网格或流动布局展示全班学生的宠物卡片。
  - **【开发改进】动态轮播与高亮**：考虑到班级人数较多（30-50人），单屏展示易拥挤，大屏支持“分页轮播”模式；当有学生刚被加减分或触发互动时，该学生的宠物卡片会自动置顶或高亮放大展示。
- **宠物卡片内容**：
  - 学生姓名 & **【产品改进】宠物昵称**（允许学生为自己的宠物起名，增强主人翁意识和情感羁绊）。
  - 宠物形象（当前进化形态及特效）
  - 当前等级 (Lv.)
  - 经验进度条 (EXP)
  - 心情进度条 (Mood)
- **交互功能**：
  - 支持触摸/点击互动。点击某只宠物时，可增加心情值，并触发AI对话聊天框。
  - *注：心情值仅增加互动趣味性，不与等级经验挂钩。*
- **全局浮窗功能**：
  - 在页面右下角设有一个类似星星/Gemini风格的小号悬浮图标。
  - 点击即可展开“AI大模型配置”弹窗，用于配置宠物对话背后的AI模型。

### 3.2 教师管理页 (教师管理页面)
- **访问限制**：需输入管理密码（初始预设为：`xxx`）方可进入。
- **核心功能**：
  - **名单管理**：支持批量上传录入、修改、删除学生名单。录取新生时，随机分配一颗不同颜色的初始蛋。
  - **【开发改进】防误触与撤销机制**：快捷加减分操作会有轻量级的Toast提示，且提供最近操作记录侧边栏，支持“撤销(Undo)”近期的误操作。
  - **快捷积分操作**：通过点击预设的行为标签，快速为单一或多个学生加减分。
  - **【产品改进】O2O 现实奖励核销**：当学生达到指定级别（如Lv.5, Lv.10），后台会产生“里程碑提醒”，教师可据此兑现现实奖励（如“免写一次作业”、“换一次座位”），形成“线上积分->线下奖励”的完整闭环。
  - **行为标签库（暂定，可随时在设置中增删）**：
    - **🟩 正向行为 (加分)**：帮助打扫卫生 (+5)、帮忙擦黑板 (+3)、写字工整/作业优秀 (+5)、主动回答问题 (+3)、好人好事 (+5)等。
    - **🟥 负向行为 (扣分)**：迟到/早退 (-3)、课上讲话/扰乱纪律 (-5)、逃避值日 (-5)、未交作业 (-5)等。

### 3.3 学生查询页 (学生查询页面)
- **定位**：供学生在平板或家长手机上单独查看详细信息。
- **核心功能**：
  - 搜索/选择自己的姓名。
  - 查看宠物的详细大图、当前等级、心情值。
  - 查看个人的“积分账单”（即最近的加扣分记录和原因），方便了解进步空间。

### 3.4 班级数据仪表盘 (仪表盘)
- **定位**：数据统计与班级整体情况反馈。
- **核心功能**：
  - 班级总积分展示。
  - 宠物等级排行榜（Top 5 或进度总览）。
  - 近期最常触发的行为统计（例如：“本周本班发生最多的好人好事”）。

## 4. 核心系统机制

### 4.1 宠物升级与“盲盒进化”系统
- **初始状态**：所有学生初始积分为0。初始宠物形态为**颜色各异的“史莱姆/小怪兽蛋”**（如红、蓝、绿、紫等不同主色调，分配时随机抽取）。
- **升级机制**：经验值累计达到阈值即可升级（例如：0-50分Lv1，50-120分Lv2...）。
- **【产品改进】教育心理学护航：降级保护机制**：为避免学生产生受挫感和内耗，**扣分（负向行为）只会扣除当前级别的EXP，绝不会导致【降级】和【失去已获得的特效】**。如果当前EXP扣到0，宠物只会表现为“生病/灰暗”状态，但不剥夺已获得的荣誉。
- **【产品改进】赛季制 (Season Reset)**：防止学霸早早满级失去动力，学渣前期落后直接摆烂。引入“学期/月度赛季”概念，每学期可进行“转生”或“收集进入图鉴”，然后从新的蛋重新开始，给每个孩子重新证明自己的机会。
- **盲盒特效（进化池）**：
  - 每次升级时，系统从特征池中随机抽取（或升级解锁）一种外观特效叠加在宠物身上。
  - **特效举例**：长出天使翅膀、浑身发光/光环、体型变大(巨大化)、长出龙角、戴上王冠、喷射星星粒子、赛博朋克风眼镜等。
  - 等级越高，特效越华丽叠加。史莱姆/怪兽蛋本身也会随着等级破壳并展现出其对应的基础怪兽形态。
  - **【开发改进】特效挂载点管理**：从美术设计角度，特效需区分“背部/背饰”、“头部/头饰”、“身体/材质”、“底部/光环”等挂载位，避免多个特效盲目叠加导致画面穿模、杂乱。

### 4.2 互动与心情系统 (AI对话大模型接入)
- **【开发改进】懒计算心情机制**：为避免服务器跑定时任务（成本高），或前端由于关闭网页导致时间停滞，**心情系统采用“时间戳差值计算法”**。数据库仅记录 `lastInteractionTime`（最后一次互动的时间戳）。大屏或用户端在渲染时，通过对比当前时间，前端自动推算展示心情衰减情况。
- **AI 赋能专属对话**：
  - 大屏或学生查询页点击宠物，将弹出对话气泡。用户的提问会发送给配置的AI大模型，模型生成专属的萌宠语气回复（回复的Prompt可内置设定：包含当前学生的姓名、近期加分情况等信息，让AI表现得像该学生的专属宠物）。
  - **【产品改进】K-12 黄金护卫 Prompt (AI安全围栏)**：系统内置的Prompt中必须包含强力约束——“你是一个面向K-12小学生的虚拟班级宠物，你的回答必须积极、阳光、不可包含任何成人、暴力、极端、讽刺内容。你需要鼓励孩子努力学习、遵守纪律，并在他们扣分时给予温暖的改进建议。”
- **完全本地化的灵活AI配置**：
  - 用户点击右下角的Logo，可打开API配置面板。
  - **可选平台**：提供快捷选项供选择：Gemini、Deepseek、ChatGPT。
  - **自定义其他模型**：提供“其他模型”选项，并支持选择接口兼容格式（OpenAI 格式 或 Anthropic 格式）。
  - **表单内容**：自定义接口地址 (Base URL)、模型名称 (Model名称)、接口密钥 (API Key)。
  - **隐私安全**：上述所有AI配置和密钥**仅保存在当前展示设备本地浏览器 (LocalStorage) 中**，不在任何服务器和数据库记录。
  - **【开发改进】跨域(CORS)策略声明**：纯前端直接调用外部大模型API可能会被浏览器拦截。系统将建议用户搭配CORS代理，或优先推荐使用开启跨域支持的模型API。

## 5. 技术与数据架构建议
- **前端页面**：React + Vite + Tailwind CSS，配合 Framer Motion 实现平滑的卡片动画和开盲盒特效。
- **后端/数据库**：
  - 采用 **Firebase Firestore** 作为实时通信与数据同步中枢，确保大屏页、管理页、查询页数据实时一致。
  - **【开发改进】监听防抖**：针对Firestore连结，接收到全班数据时采用优化过的React渲染机制（如 `React.memo`），避免不必要的全局Re-render，保障大屏长驻运行的性能。
- **第三方API调用**：AI对话由前端直接调用（基于本地保存的URL和Key），经过强制安全Prompt封装后发送，轻量且注重隐私。

## 6. UI/UX 设计规范 (Duolingo 多邻国风格)
本项目整体采用 **“果冻感”与微质感毛玻璃 (Playful & Glassmorphism)** 的现代化教育应用风格。它既不会过于低幼，又能为师生提供极强的正向情绪价值和治愈感。

### 6.1 视觉特征 (UI)
- **多巴胺色彩体系**：使用高明度、高饱和的“糖果色”（如明亮黄、薄荷绿、天空蓝），搭配纯白或浅灰背景，保持视觉呼吸感。
- **极致圆角与果冻感 (Squircle)**：卡片、按钮、头像框和进度条都必须使用大圆角（完全消除锐利的直角），营造出Q弹软萌的舒适视觉感。
- **立体层次与微交互外观**：
  - 大屏展示界面加入轻微的毛玻璃效果（Glassmorphism）和淡淡的弥散色彩背景，令整体更具现代科技感。
  - 交互元素（特别是按钮和宠物卡片）采用略带厚度的“双层阴影”或微高光，打造一种仿佛真实材质的立体按压感。

### 6.2 交互物理反馈 (UX)
- **弹簧力学动画 (Spring Physics)**：
  - 摒弃死板的线性过渡。所有页面切换、按钮悬停、点击等动作，应当产生犹如弹簧般的“下沉并回弹”物理响应。
- **微动画与情感化反馈**：
  - **粒子特效**：给学生加分或成功升级时，必须弹出有视觉冲击力的金币、星星或彩带飞出的爆炸粒子特效（Confetti/Particle）。
  - **宠物情绪表现**：给宠物喂食或互动时，宠物本身会轻微“发抖”或弹跳；心情差时，卡片采用灰暗或下垂的视觉暗示。
  - 这能建立强烈的留存钩子和养成欲，让学生每天期待看到大屏。

### 6.3 实现路径策略 (Technical Implementation)
- **CSS 框架深度定制**：在 `Tailwind CSS` 中扩展自定义的糖果色主题色板，并写入专属的 `box-shadow` 实用类（例如实现按键厚度感的底部阴影）。
- **动画引擎引入**：全面融入 `Framer Motion`，将绝大部分交互动画的 `transition` 设置为包含 `stiffness`（刚度） 和 `damping`（阻尼） 的 `type: "spring"` 弹性动画。
- **复合视觉层叠管理**：升级后获得的特殊盲盒配件（如皇冠、小翅膀等），将以处理为高透明背景的无损素材（WebP/PNG或SVG）的方式，利用纯 CSS + `z-index` 进行绝对定位堆叠覆盖，保持高清且易于拓展。
- **粒子发生器组件**：集成库实现即插即用的粒子发生器，在关键教师操作和学生荣誉时刻触发强情绪反馈。
